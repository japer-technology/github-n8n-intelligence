# Questions

> [Index](./index.md) · [README](../../README.md)

Six foundational questions define GitHub Minimum Intelligence. Each examines the project from a different angle—together they form a complete picture of what repository-native intelligence means and why it matters.

| Question | Summary |
|----------|---------|
| **What?** | Defines GitHub n8n Intelligence as a repository-native workflow automation platform — n8n's 400+ integration nodes running entirely on GitHub infrastructure, with an AI agent for conversational interaction. |
| **Who?** | Explores identity and accountability: who speaks, who executes, who remembers, and who governs when the repository itself becomes the automation platform. |
| **When?** | Examines the temporal architecture: how Git commits replace ephemeral sessions, making workflow history durable, trust auditable, and automation resilient across time. |
| **Where?** | Maps where intelligence physically lives — runtime in GitHub Actions, workflows in versioned JSON files, identity in checked-in markdown, and authorization in existing repository permissions. |
| **How?** | Walks through the implementation — issues as conversational input, n8n as execution engine, Actions as compute runtime, an LLM as reasoning substrate, and Git commits as durable memory. |
| **How Much?** | Asks how much automation a repository can hold, and argues the ceiling is workflow complexity, not infrastructure — n8n's 400+ nodes provide virtually unlimited integration capability. |
